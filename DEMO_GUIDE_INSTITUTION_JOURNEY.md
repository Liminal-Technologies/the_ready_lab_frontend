**Step-by-Step Fix:**

#### Issue 2: SSO/SAML Configuration Errors
**Symptoms:**

- Certificate validation fails
- Attributes not mapping correctly

**Root Causes:**

- Attribute claim names don't match
- Clock skew between systems
- Firewall blocking SAML traffic

**Step-by-Step Fix:**

**For Azure AD:**
1. **Verify URLs match exactly:**
   - Entity ID: `https://yourinstitution.thereadylab.com/auth/saml/metadata`

2. **Re-download certificate:**
   - Azure Portal â†’ Enterprise Apps â†’ Your App â†’ Single sign-on

   - Upload to The Ready Lab settings
3. **Check attribute mappings:**
   - Email claim: `http://schemas.xmlsoap.org/ws/2005/05/identity/claims/emailaddress`
   - Name claim: `http://schemas.xmlsoap.org/ws/2005/05/identity/claims/name`
   - Test with known user
4. **Verify certificate expiration:**
   - Check cert expiry date (Settings â†’ SSO â†’ Certificate expires: XX/XX/XXXX)
   - Renew if <30 days remaining
5. **Test connection:**
   - Use "Test Connection" button
   - Review detailed error logs
   - Share logs with support if needed

**For Okta:**
1. Ensure Okta app uses SAML 2.0 (not WS-Fed)
2. Set Sign-On URL to correct environment
3. Map firstName and lastName attributes
4. Enable "Use Okta for SSO" in user settings

**Common Azure AD Error Messages:**
- **"AADSTS50011: Reply URL mismatch"** â†’ ACS URL in Azure doesn't match The Ready Lab's

### Common Demo Pitfalls & How to Avoid Them

#### Pitfall 1: Skipping the "Why"
**Mistake:** Jumping straight into feature demo without establishing pain points

**Fix:**
- Start every demo with discovery: "What challenges are you facing with your current LMS?"
- Listen for keywords: "low completion rates," "manual processes," "expensive," "slow to deploy"
- Tailor demo to address THEIR specific pain points
- Use case studies from similar institutions

---

#### Pitfall 2: Feature Overload
**Mistake:** Showing every feature in 60 minutes â†’ overwhelms prospect

**Fix:**
- **Pick 3-4 key features** based on their pain points
- Example for completion rate issue: Focus on AI coaching, micro-learning, mobile UX
- Example for admin efficiency: Focus on CSV upload, automated reporting, bulk assignment
- Say: "We have 50+ features, but let me show you the 3 that matter most for your use case..."

---

#### Pitfall 3: Not Addressing Budget Early
**Mistake:** Waiting until end of demo to discuss pricing â†’ sticker shock kills momentum

**Fix:**
- Address budget in first 10 minutes: "To make sure we're aligned, what's your budget range for this?"
- If they say "$5K/month," you know $9.5K/month won't work â†’ pivot to smaller cohort or phased rollout
- Frame pricing in context of ROI: "$9,500/month for 1,000 users is $9.50 per userâ€”less than the cost of a textbook"

---

#### Pitfall 4: Ignoring the Technical Buyer
**Mistake:** Demo to program manager, but IT/security team kills deal later

**Fix:**
- Ask early: "Who else needs to be involved in this decision? IT? Legal? Procurement?"
- Invite IT to demo or schedule separate technical deep-dive
- Proactively share security docs (SOC 2 report, data flow diagrams)
- Anticipate IT objections: SSO complexity, data privacy, uptime SLA

---

#### Pitfall 5: No Clear Next Steps
**Mistake:** Great demo, but no follow-up plan â†’ deal stalls

**Fix:**
- End every demo with next steps:
  - "I'll send pricing proposal by EOD tomorrow"
  - "Let's schedule technical review with your IT team for next week"
  - "I'll connect you with 3 reference clients this week"
- Get commitment: "If the pilot goes well, when would you want to launch for fall semester?"
- Calendar invite before they leave: Schedule follow-up call
